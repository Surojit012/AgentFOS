# Integration Patterns

AgentFOS can be integrated at the orchestration level or at the skill level.

## Full Pipeline

Use `/agent` for natural-language orchestration.

```text
Agent → /agent → parse → opportunity → risk → policy → allocation
```

## Structured Pipeline

Use individual endpoints when your system already controls state.

```text
/policy → /opportunity → /risk → /allocate
```

## Risk-Only Integration

Use `/risk` when another system only needs deterministic protocol scoring.

```bash
curl -sS -X POST https://agentfos.onrender.com/risk \
  -H "Content-Type: application/json" \
  -d '{"protocol":"ondo"}'
```

## Pharos Proof Integration

Use `/allocate` when you need portfolio output plus optional on-chain proof.

The write is demo-safe: if Pharos config, RPC, private key, or `cast` is unavailable, the allocation still returns and the on-chain status explains the failure.
