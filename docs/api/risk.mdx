# POST /risk

Scores a protocol deterministically with Risk Engine v2.

The response remains backward compatible. Existing consumers can keep reading `risk_score`, `risk_level`, `factors`, and `recommendation`.

## Request

```json
{
  "protocol": "ondo"
}
```

## Request Fields

| Field | Type | Required | Description |
|---|---|---:|---|
| `protocol` | string | Yes | Protocol key such as `ondo`, `zoth`, `maple`, `centrifuge`, `backed`, or `opentrade` |

## Risk Engine v2 Fields

| Field | Description |
|---|---|
| `rating` | Display label for the numeric score |
| `overall_confidence` | `high`, `medium`, or `low` |
| `breakdown` | Dimension scores and provenance |
| `scored_at` | UTC scoring timestamp |
| `schema_version` | Risk schema version, currently `2.0` |

## Breakdown Dimensions

| Key | Max |
|---|---:|
| `liquidity` | 25 |
| `maturity` | 20 |
| `transparency` | 20 |
| `concentration` | 20 |
| `yield_sustainability` | 15 |

## Provenance Enum

| Value | Meaning |
|---|---|
| `live` | Fresh primary-source data was used |
| `degraded` | Fallback, stale, partial, or static data was used |
| `insufficient` | No usable data existed; a documented floor score was applied |
